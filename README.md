# あなたはだーれ（オンライン対戦版）

ゲームマーケット作品「偏見プロフィール」を原案にした、Node.js + Socket.IOによるオンライン対戦実装（MVP）。

## 現状のスコープ

- 人物カードは **ダミー画像**（placehold.co、40枚）。ロジック優先で、実際の顔写真は後で差し替え可能。
- プロフィールお題は24種類をコードに直書き（`server.js` の `PROFILE_PROMPTS`）。
- 1部屋＝1ゲーム。プレイヤー全員が1回ずつ親になったら終了。
- 再接続時の状態復元は未実装（切断＝退席扱い）。

## ゲームの流れ（実装済みロジック）

1. ホストが部屋作成 → 他プレイヤーが4桁の部屋番号で入室（ロビー、最低3人）。
2. ホストが開始 → 親をランダム順で1人ずつ回す。
3. 記入フェーズ：子プレイヤー（親以外）に3つのお題を割り振り、お題人物の画像を見ながら偏見全開でテキスト回答。親には何も見えない（待機画面）。
4. 全員の記入が揃うと、お題人物＋ランダムな4人をシャッフルして5枚を親に公開（正解は隠す）。
5. 各子プレイヤーは自分の担当ヒントを好きなタイミングで公開（`reveal_profile`）。
6. 3つ全部公開されると、親が5枚から本人を選択（`submit_guess`）。正解なら親に1枚（1点）。
7. 結果発表 → ホストが「次のラウンドへ」→ 全員が親を経験したら最終結果画面。

## セットアップ

```bash
npm install
npm start
```

`http://localhost:3000` にアクセス。同じネットワーク内の別端末からは `http://<自分のPCのIP>:3000` でアクセス可能。

## 簡易プレイテスト

`test/playtest.js` に、3人のダミークライアントでロビー〜ゲーム終了までを自動で流す動作確認スクリプトがあります。

```bash
node server.js &
node test/playtest.js
```

## デプロイ（Railway / Render 想定）

- `PORT` 環境変数を読むようになっているので、そのままRailway/Renderにデプロイ可能。
- 状態はメモリ内（Map）保持のみ。複数インスタンス（オートスケール）構成にすると部屋がインスタンス間で共有されないので注意（単一インスタンスで運用するか、後日Redis等の外部ストア対応が必要）。
